# 셀러 편의성 UX 개선 리포트 — 2026-05-30

짐스캐너 SELLER (B2B) 전체 기능을 **바쁜 마켓 셀러(월 100~500건 처리) 관점의 편의성**으로 심층 검토한 개선 제안서.

- **검토 범위**: 5개 기능 영역(주문 / 매입·SKU·영수증 / 배대지·양식·ETA / 재무·정산·부가세 / 온보딩·대시보드·설정) 전 페이지·컴포넌트
- **방법**: 영역별 병렬 리뷰 에이전트가 실제 코드를 셀러 일상 워크플로우 기준으로 평가 → 우선순위·근거(file:line)·공수(S/M/L) 산출 → 교차 테마 종합
- **성격**: 버그가 아닌 **개선 제안**. 일부는 구현 전 사용자 검증 권장(공수/우선순위는 추정).
- 총 60개 제안 → 본 리포트는 **교차 테마 6개 + 영역별 핵심 + 우선순위 로드맵**으로 종합.

---

## 0. 한눈에 — 가장 큰 6개 교차 테마

영역을 가로지르며 반복 등장한 패턴. 이게 제품 편의성의 핵심 레버.

| 테마 | 한 줄 요약 | 등장 영역 |
|---|---|---|
| **T1. 경고를 "행동 시점"에 띄워라** | 마진 손실·카드 한도·배대지 양식 누락 필드를 대시보드에만 두지 말고 *주문 입력/내보내기 그 화면에서* 경고 | 주문·재무·배대지 |
| **T2. 엑셀 패리티 (bulk·fill-down)** | 셀러는 엑셀러. fill-down, 행 복제, 일괄 매칭/해제, bulk 인라인 SKU 가 없어 재타이핑 발생 | 주문·매입 |
| **T3. 반복 작업 가속 (recents·favorites·sticky)** | 같은 구매자·같은 SKU·같은 배대지를 매번 다시 고름. 최근/즐겨찾기/마지막값 기억으로 클릭 제거 | 주문·매입·배대지 |
| **T4. 플래그를 "클릭 가능한 액션"으로** | 정산 ±15% 주의·저가 경쟁·통관 실패·ETA 지연을 *보여만 주고* 해당 주문으로 가는 링크가 없음 | 정산·분석·ETA·컴플라이언스 |
| **T5. 신규 셀러 time-to-value** | 인증 다음 단계 CTA가 묻혀있고, 이메일 인증 흐름 불명확, 온보딩 트리거 너무 좁음 | 온보딩 |
| **T6. 설정 IA·모바일** | /settings 18카드 중 11개 "준비 중"이 섞여 가용 기능을 못 찾음. 모바일 사이드바가 화면 절반 | 설정 |

## 1. 🥇 우선순위 로드맵 (impact ÷ effort)

### Tier 1 — 즉효 Quick Wins (S 공수, 높은 체감)
1. **주문 입력 화면 카드 한도 경고 배너** — `/orders/new` 상단에 80%+ 카드 있으면 경고 (T1) · S
2. **fill-down / 행 복제** — bulk 그리드에 Ctrl+D 채우기 + 행 복제 (T2) · S~M
3. **신규 셀러 "다음 인증 단계" CTA를 상태 배너에 통합** — 묻힌 카드 대신 단일 CTA (T5) · S
4. **행동 큐 노이즈 감축** — 1건이어도 큰 섹션 띄우는 것 → 3건+ 또는 인라인 칩 (T6/대시보드) · S
5. **정산 ±15% "주의" 배지 클릭 → 해당 주문 목록** (T4) · S
6. **분석 저가 경쟁 행 하이라이트** — 시장 중앙값 −10% 행 amber 배경 (T4) · S
7. **SKU 즐겨찾기/공급처·통화 칩을 picker·quick-pick 결과에 표시** (T3) · S
8. **빈 상태에 맥락 추가** — 알림/지원 빈 페이지에 "어떻게 생기나 + CTA" (T5) · S

### Tier 2 — 핵심 플로우 개선 (M 공수, 높은 impact)
9. **point-of-action 마진 손실 경고** — 주문 입력 시 "이 SKU 현재가 손실" (T1) · M
10. **bulk 인라인 SKU 피커** — `/orders/bulk` product_name 옆 SKU 선택 → 공급처·통화·가격 자동 채움 (T2/T3) · M
11. **영수증 일괄 매칭 (점수≥90 / 70–89 분리 일괄 수락)** — `/imports` 헤더에 bulk accept (T2) · M
12. **sticky 배대지** — 새 주문 시 마지막/최다 사용 배대지 pre-select (T3) · M
13. **양식 내보내기 사전 readiness 표시** — 주문 상세에 "양식 필드 4/5, [전화] 채우면 변환 가능" (T1/배대지) · M
14. **최근 구매자 quick-pick** — `/orders/new` 상단 최근 5명 클릭 자동 채움 (T3) · M
15. **이메일 인증 전용 화면** — step-2 자동 진행 대신 "메일 확인하세요 + 재전송" (T5) · M
16. **CSV 라인별 마진 컬럼** — 부가세 export 에 라인 환산원가·마진 추가(세무사 제출용) (T4/재무) · M

### Tier 3 — 구조·고급 (M~L, 셋업·고볼륨 사용자)
17. 자동 매칭 임계값 설정(셀러가 70/85 선택) · M
18. 영수증↔주문 라인 side-by-side 검증 뷰 · M
19. /settings "가용 / 준비중" 탭 분리 + 모바일 아이콘 사이드바 · M
20. 배대지별 무료 보관일 설정(현재 7일 상수) + 보관비 경고 정확도 · S
21. picker 내 인라인 SKU 빠른 등록 · L
22. 정산/컴플라이언스 실패 드릴다운 + 재시도 · M

## 2. 영역별 핵심 findings

### A. 주문 관리·입력
- **[HIGH] bulk 그리드 엑셀 패리티 부재** — fill-down·행 복제·인라인 SKU 피커 없음. 같은 공급처/통화 20행을 일일이 입력. (BulkOrderClient)
- **[HIGH] 검증 실패 시 첫 오류 셀 auto-focus 없음** — 50행 중 어디가 틀렸는지 수동 스캔.
- **[HIGH] 반복 구매자 재입력** — 매주 같은 구매자도 이름·전화·주소·통관코드 매번 타이핑. recents/template 없음.
- **[MED] 목록에 "영수증 미매칭" 빠른 필터 없음** — `receipt_count` 표시는 하나 `=0` 필터·배지 없음.
- **[MED] 운송장 paste 후 목록 미반영** — tracking-paste 결과에서 일괄 in_transit 전이 버튼 부재.
- **[MED] 매칭 가격 차이 경고 단일 임계(≥5%)·amber** — <5/5–10/>10% 색 구분 + variance 리포트 뷰 권장.
- **[MED] bulk-delete/undo 부재** — 잘못 만든 20건 개별 삭제.

### B. 해외 매입·SKU·영수증 매칭
- **[HIGH] 영수증 일괄 매칭 부재(개별 목록)** — `OrderMatchingClient` 엔 bulk-apply 있으나 `/imports` 개별 목록엔 없음. 20건=20 클릭 사이클.
- **[HIGH] SKU 마스터에 재매입 빈도·최근일 미표시** — "가장 많이 재구매" 정렬 불가, 즐겨찾기 수동.
- **[MED] 영수증 검색 모달이 공급처 사전필터 안 함** — 아마존 JP 영수증인데 전 마켓 주문 다 보여줌.
- **[MED] picker 즐겨찾기 시인성 약함** — 별 아이콘 작음, 배지/구분선 없음.
- **[MED] "이미 매칭됨" 패널티에 어느 영수증인지 링크 없음** — 해제하러 못 감.
- **[MED] 일괄 해제(bulk-unmatch) 없음** · **[LOW] picker 인라인 SKU 빠른등록 없음**(컨텍스트 스위치).

### C. 배대지·양식 변환·ETA·정산
- **[HIGH] 새 주문 시 배대지 미리 선택 안 됨** — 기본값이 마켓 감지 *이후* 채워져 100번째 주문도 수동 선택.
- **[HIGH] 양식 변환 필드 누락을 모달 열어야 발견** — 주문 상세에 사전 readiness 표시 없음.
- **[MED] bulk 내보내기 "수취인 묶음 절감"이 모달 들어가야 보임** — 목록서 사전 안내 X.
- **[MED] ETA 지연 목록에서 바로 액션(변환/상태) 링크 없음.**
- **[MED] 정산 ±15% 주의 배지 드릴다운 없음.**
- **[MED] 보관기간 기본 7일 상수** — 배대지별 정책 설정 불가 → 오경보/누락.
- **[MED] transit override 국가 자유입력** — 'USA' 입력 시 silent OTHER fallback. 드롭다운 권장.

### D. 재무·정산·부가세·분석
- **[HIGH] 마진 손실이 주문 입력 시점에 안 뜸** — 대시보드에만. 손실 주문을 모르고 기록.
- **[HIGH] 환불 라인 선택 시 원가·마진 컨텍스트 부족** — 중복 상품 시 어느 라인인지 분간 어려움.
- **[MED] 카드 한도 경고가 설정 페이지에만** — 주문 입력 시 push 필요.
- **[MED] 부가세 CSV 라인별 마진 컬럼 없음** — 합계만. 세무사용 라인 단위 필요.
- **[MED] 분석 저가 경쟁 행 미하이라이트** · **[MED] 컴플라이언스 발송 실패 드릴다운/재시도 없음.**
- **[LOW] 부가세 CSV 커스텀 기간 UI 없음**(프리셋만) · **추천 페이지 환율 실패 시 fallback 없음.**

### E. 온보딩·대시보드·설정·알림
- **[HIGH] 신규 셀러 인증 진행 카드가 노이즈에 묻힘** — 다음 액션 3클릭 거리.
- **[HIGH] 이메일 인증 흐름 불명확** — step-1 후 자동 step-2, 미인증 시 generic 에러.
- **[HIGH] 온보딩 모달 트리거 과협소** — `주문0 AND SKU0` 만. SKU 1개 만든 신규는 가이드 못 봄.
- **[MED] /settings 18카드 중 11개 "준비중" 혼재** — 가용 기능 발견성 저하. 탭 분리 권장.
- **[MED] 대시보드 행동 큐 과도** — 1건이어도 큰 섹션.
- **[MED] health score 가 day1 스냅샷 없으면 아예 안 보임** — skeleton/실시간 미리보기 권장.
- **[MED] 모바일 사이드바 260px** — 화면 절반 차지. 아이콘 모드 권장.
- **[MED] 알림/지원 빈 상태 맥락 부족** · **[HIGH] 정지 계정 안내 빈약**(사유·해결시간·원클릭 티켓 없음).

## 3. 권고

1. **Tier 1 (Quick Wins 8개)** 먼저 — 모두 S 공수, 셀러 일상에서 매일 체감. 특히 T1(행동 시점 경고)·T3(반복 가속)은 클릭 수를 직접 줄여 "제품이 똑똑해졌다"는 인상.
2. **Tier 2 의 #9·#10·#11·#12** 가 핵심 플로우(주문 입력·매칭·배대지 선택)의 재타이핑을 제거 — 제품 헤드라인 가치(5분→30초)를 실제로 끌어올림.
3. 구현 전 **#16 (CSV 라인 마진)·#13 (readiness)** 등은 기존 export/모달 로직 재사용이라 회귀 위험 낮음 — 우선 착수 후보.
4. 본 리포트의 file:line 은 검토 시점 근거이며, 일부 제안(공수·우선순위)은 구현 착수 전 1차 검증 권장.
